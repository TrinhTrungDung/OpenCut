%%{init: {'theme': 'dark'}}%% sequenceDiagram participant U as User participant PM as PlaybackManager participant RAF as RAF Loop participant VP as VideoElementPool participant CR as CanvasRenderer U->>PM: play() PM->>PM: playStartWall = now() PM->>VP: startVideoElements() VP->>VP: el.currentTime = sourceTime VP->>VP: el.play() loop Every Frame (~16ms) RAF->>PM: updateTime() PM->>PM: elapsed = now() - playStartWall PM->>PM: currentTime = startTime + elapsed PM->>CR: notifyTime(currentTime) CR->>CR: render scene graph CR->>VP: drawImage(video element) end U->>PM: pause() PM->>VP: pauseAll()

Why audio transitions are seamless: Web Audio API pre-schedules buffers 3s ahead. No cold-start latency like <video> elements.
